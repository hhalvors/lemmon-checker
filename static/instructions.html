Instructions for lemmon-checker

This tool checks Lemmon-style natural deduction proofs entered in pipe format.

1. Pipe format

Each proof line is written as:

REFERENCES|LINE_NUMBER|FORMULA|JUSTIFICATION
  • REFERENCES: Comma-separated list of dependency numbers, or empty for none.
  • LINE_NUMBER: The current line number in the proof.
  • FORMULA: A logical formula using the connectives below.
  • JUSTIFICATION: Rule name (and possibly references) explaining the step.

Example

1|1|P→∀xF(x)|A
2|2|P|A
1,2|3|∀xF(x)|1,2 MP
1,2|4|F(a)|3 ∀E
1|5|P→F(a)|2,4 CP
1|6|∀x(P→F(x))|5 ∀I

2. Logical connectives

Use the following symbols in formulas:

  • ¬ – negation
  • ∧ – conjunction (and)
  • ∨ – disjunction (or)
  • → – conditional (if … then)
  • ↔ – biconditional (if and only if)
  • ∀x – universal quantifier
  • ∃x – existential quantifier

Predicates are uppercase letters followed by terms, e.g. F(a) or Rxy. Constants are lowercase letters (a, b, c, …), and variables are also lowercase.

3. Common justification abbreviations

  • A – Assumption
  • MP – Modus Ponens
  • CP – Conditional Proof
  • ∀E – Universal Elimination
  • ∀I – Universal Introduction
  • ∃E – Existential Elimination
  • ∃I – Existential Introduction
  • RAA – Reductio ad Absurdum

4. Tips

  • Each formula must be well-formed according to the syntax rules above.
  • Whitespace is ignored in most places.
  • Make sure dependencies and line numbers match your intended scope.
